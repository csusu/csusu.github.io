SQLServer Single User Mode

SQL Server 单用户模式解决访问权限问题

1. 问题描述

本机操作系统Windows 2012 Professional, 安装了SQL SERVER 2012 EXPRESS，只开启了Windows身份验证，在非域环境下运行正常。
最近因为公司AD域升级，要求所有机器加入Windows Server 2012 R2域环境。本机加入域环境后其他程序均正常，但是运行SQL Server Management Studio(ssms)后，
首先是无法连接，更改到正确的机器名\实例名并使用对应的域用户账号后可以正常连接数据库，但是没有任何管理权限，原先建立的数据库也无法访问，错误提示是：

用户没有执行此操作的权限或者对象不存在。(与原文可能不一致，大概是这个意思)

由于没有启用SQL Server身份验证，因此sa账号无法登陆，尝试启用sa账号也报同样的错误。

2.问题分析

通过SSMS的错误提示，分析最可能的原因是SQL Server账号访问权限问题。因为加域后，新的域账号没有对应访问权限。
尝试使用本地管理员账号登陆操作系统后再连接数据库，还是同样的问题。根据网上的搜索结果提示可以使用
psexec工具执行下面的指令：

PsExec.exe -i -s ‘C:\Program Files (x86)\Microsoft SQL Server\110\Tools\Binn\ManagementStudio\Ssms.exe’

获得system权限运行ssms，不过仍然无法解决问题。

3.问题解决

显然通过用各种Windows系统账号(如administrator、system等)运行ssms是无法解决问题的。关键还是要能够启用数据库的sa账号。
因此考虑使用SQL Server的维护模式来启用sa账号。

  • Step 1 关闭SQL SERVER (EXPRESS)服务。管理员命令行下执行下列命令：

    NET STOP MSSQL$SQLEXPRESS

  • Step 2 以单用户模式启动SQL SERVER (EXPRESS)服务：

    NET START MSSQL$SQLEXPRESS /M

  • Step 3 管理员身份运行ssms工具，并新建查询。注意此时是单用户模式，不能同时新建查询和连接服务器。

  • Step 4 启用sa账号：

    alter login sa enabled

  • Step 5 以多用户模式重启SQL Server服务

    NET STOP MSSQL$SQLEXPRESS
    NET START MSSQL$SQLEXPRESS

此时已经实现了sa账号的启用，但是却不能用sa账号登陆，因为服务器还没有设置为Windows和SQLServer混合登陆模式，修改注册表

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Microsoft SQL Server\MSSQL11.SQLEXPRESS\MSSQLServer

路径下的LoginMode值从1改为2。然后重启SQL Server服务，使用sa账号登陆后将当前域账号设置为sysadmin。
